# Логічний редактор v8 — короткий звіт

## Зроблено

- Дроти за замовчуванням чорні; червоний колір використовується лише для помилки або логічного конфлікту. Бейджі `X/Z/0/1` на дротах прибрано.
- Додано ортогональне трасування H/V, квадратні ручки bend points, перетягування сегментів без відриву від пінів, snap-to-grid і збереження `bendPoints`/`color` у JSON.
- Під час руху дроту або ручки оновлюється конкретний SVG path; повний canvas не перемальовується на кожен pointer move.
- Подвійний клік по дроту відкриває стабільне меню кольору з палітрою та color picker. Подвійний клік по MUX, DECODER, FULL_ADDER і CUSTOM відкриває внутрішню схему.
- Додано ANSI-символи вентилів та окремі схемні символи для MUX, DEC, SUM, DFF, CTR, CLOCK, DIP, HEX, 7SEG, ADuC і CUSTOM. Такими самими SVG-прев’ю заповнено верхню панель.
- Інструменти вибору, дроту й видалення зроблено компактними. Додано fullscreen, компактний popover властивостей і максимально велику область canvas без правої панелі.
- Реалізовано Ctrl+C/V/X, Ctrl+Z, Ctrl+Y, Ctrl+Shift+Z, Ctrl+A, Delete, Esc, V і W.
- Додано Shift-click, рамкове multi-select, виділення дротів/елементів, hover-підсвітку допустимих пінів та перевірку input-input/output-output.
- Старі JSON-проєкти без `bendPoints` і `color` лишаються сумісними. `evaluateCircuit` не змінювався.

## Перевірено

- `npm ci` — успішно.
- `npm run build` — успішно.
- `npm run test:logic` — 155 базових + 6 нових routing-перевірок.
- `npm test` — успішно: ASM, C51, language extensions, completions, WASM runtime, осцилограф і логічні схеми.

## Обмеження

- Ручки редагують ортогональні H/V маршрути; довільні діагональні сегменти та електричні junction-вузли кількох дротів не додаються.
- Буфер Ctrl+C/V є внутрішнім буфером редактора і не переноситься між вкладками або після перезавантаження сторінки.
- Автотести перевіряють модель, сумісність JSON та обчислення; pointer-жести UI не покриті окремим browser E2E-набором.
